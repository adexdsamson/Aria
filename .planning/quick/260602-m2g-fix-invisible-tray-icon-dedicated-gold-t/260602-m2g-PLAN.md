# Quick Task 260602-m2g — Fix invisible system-tray icon

## Problem
Aria's Windows system-tray icon renders as a blank square. Root cause: the tray
assets (`build/tray-*.ico` / `*Template.png`) were rasterised from `build/icon.svg`
— the **app-launcher tile** (ivory `#FAFAF8` squircle + thin dark serif "A"). At 16px
on a dark tray that low-contrast light tile reads as a featureless/blank square.
Secondary defects: macOS Template PNGs were generated from the *coloured* art (a
Template must be black+alpha only), and `tray-icon.ico` == `tray-icon-badged.ico`
byte-for-byte (the gold dot never baked).

## Fix
Give the tray its own glyph artwork, separate from the app tile:
- `build/tray-glyph.svg` — **gold (`#B8860B`) squircle + bold ivory "A"** → Windows `tray-icon.ico` (high contrast, visible on dark AND light trays).
- `build/tray-glyph-badged.svg` — same + ivory activity dot → `tray-icon-badged.ico`.
- `build/tray-glyph-template.svg` — **monochrome black "A"** on transparent → `tray-iconTemplate.png` (+@2x); AppKit auto-inverts on the menu bar.
- `build/tray-glyph-badged-template.svg` — black "A" + black dot → `tray-iconBadgedTemplate.png` (+@2x).
- `scripts/build-tray-icons.mjs` — source the `.ico` from the colour glyphs and the Template PNGs from the monochrome glyphs (was sourcing everything from `icon.svg`).
- Regenerate via `pnpm run icons:tray`.

`src/main/tray/icons.ts` already loads `.ico` on win32 and `*Template.png` on darwin — no loader change needed.

## Acceptance
- Tray icon visible + recognisable (gold "A" tile) on the Windows tray.
- `tray-icon.ico` ≠ `tray-icon-badged.ico` (badge bakes).
- macOS Template PNGs are monochrome (black+alpha).
- Verified via 64px preview render (gold tile + ivory A) before commit; live confirm on app restart (tray loads at bootstrap — no HMR).
