  In VoicePTTButton.tsx:
  1. Add a cloudReadyRef (useRef<boolean | null>(null)) alongside modelReadyRef to cache the cloud check result separately. Cache invalidation follows the same pattern: null = unchecked.

  2. Add an async helper checkCloudEnabled(): Promise<boolean> that:
     - Returns cloudReadyRef.current if not null (cached).
     - Calls getIpc()?.voiceGetPrefs(). If IPC is absent or voiceGetPrefs is undefined on the IPC object, returns false (no cloud = fall through to local check).
     - On success, reads result.useCloud; caches and returns that boolean.
     - On any thrown error, caches false and returns false (do not fail-open for cloud — fall through to local model check instead).

  3. Add an async helper checkReadyOrCloud(): Promise<boolean> that:
     - First awaits checkCloudEnabled(). If true, returns true immediately (PTT is allowed regardless of local model state).
     - Otherwise awaits the existing checkModelReady() and returns its result.

  4. In attemptPttStart(): replace the two call sites of checkModelReady() (the async path's .then() chain and the null-check branch that starts the async check) so that the async check calls checkReadyOrCloud() instead of checkModelReady(). The fast-path synchronous check on modelReadyRef.current === false must ALSO account for the cloud case: only block on false if cloudReadyRef.current !== true. Concretely:
     - At the top of attemptPttStart(), before the modelReadyRef.current === false fast-path, add: if (cloudReadyRef.current === true) { session.setVadMode(vadMode); session.startTurn(); return; }
     - Keep the modelReadyRef.current === false early-return (show modal) only when cloudReadyRef.current is NOT true.
     - Keep the modelReadyRef.current === true fast-path as-is.
     - In the null (not yet checked) branch: replace the checkModelReady() call with checkReadyOrCloud().

  Note on caching: cloudReadyRef caches the first voiceGetPrefs() result for the session (same lifetime as modelReadyRef). If the user changes cloud setting mid-session without remounting, the cache is stale — acceptable for v1 since toggling cloud while PTT is hot is an edge case. Document this with a one-line comment.
