# Quick Task 260602-e4h: Production app branding (icon + name)

## Problem

Packaged build showed the **default Electron logo** and a generic **"electron"**
Windows taskbar identity instead of the Aria brand monogram + name.

## Root causes & fixes

### 1. No raster app icons → Electron default logo
Only `build/icon.svg` existed; electron-builder had no `icon` configured, so it
embedded its default icon.
- New `scripts/build-app-icons.mjs` (sharp + to-ico, mirrors `build-tray-icons.mjs`)
  + `icons:app` npm script → `build/icon.png` (1024) + `build/icon.ico`
  (16/24/32/48/64/128/256).
- `win.icon=build/icon.ico`, `mac.icon=build/icon.png` (electron-builder derives
  `.icns` on the mac runner), `linux.icon=build/icon.png`.

### 2. resolveBrandIcon pointed at unpackaged SVG → default window icon
`build/icon.svg` was not in packaged `files[]` (never in the asar) and Electron
can't rasterise SVG for a Win/macOS window icon.
- Repointed `resolveBrandIcon()` at the raster: `process.resourcesPath` in
  packaged builds (`icon.ico` on Windows, `icon.png` elsewhere), `build/` in dev.
- Added `icon.ico` + `icon.png` to `extraResources` so they land in `resources/`.

### 3. AUMID mismatch → generic "electron" taskbar identity
`app.setAppUserModelId('com.aria.app')` ≠ electron-builder `appId`
`com.aria.desktop` (the AUMID the NSIS shortcut carries). Windows couldn't bind
the running app to the installed shortcut → fell back to generic identity.
- Changed AUMID to `com.aria.desktop`. Builds already published under that appId,
  so match code → appId (not the reverse).

## must_haves
- **truths:** packaged app shows the Aria monogram (taskbar + window + installer)
  and binds to the `com.aria.desktop` Windows identity.
- **artifacts:** `build/icon.png`, `build/icon.ico`, `scripts/build-app-icons.mjs`,
  `package.json` (icon config + extraResources + icons:app), `src/main/index.ts`
  (resolveBrandIcon + AUMID).
- **key_links:** `resolveBrandIcon` in `src/main/index.ts`; `build.win/mac/linux.icon`
  + `extraResources` in `package.json`.
