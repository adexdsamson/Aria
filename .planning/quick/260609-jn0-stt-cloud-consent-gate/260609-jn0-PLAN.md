<objective>
Fix the STT-audio cloud-routing gate so cloud STT fires when cloud audio is enabled.

Root cause: VOICE_FEED_AUDIO evaluates `shouldUseCloud('', llmQueue, prefs.useCloud)`. Empty string
hits `classify('')` → confidence < 0.6 → always returns false → cloud STT never fires even with
useCloud=true. The shouldUseCloud gate is correct for the LLM-answer leg (post-transcript context
sensitivity) but wrong for the audio leg (audio precedes transcript — nothing to classify yet).

Fix: Replace the shouldUseCloud call with a pure consent-and-availability check
(`prefs.useCloud === true && cloudSttResolved != null && wavUtilsResolved != null`).
shouldUseCloud() stays intact and exported for the answer leg; it is NOT removed.

Purpose: Cloud STT (whisper-1) must fire when the user has opted in via the VoiceSection
disclosure modal (D-14 consent). The current bug silently falls back to local every time.

Output: Fixed voice.ts + rewritten spec with prefs.useCloud-gated test cases.
</objective>
