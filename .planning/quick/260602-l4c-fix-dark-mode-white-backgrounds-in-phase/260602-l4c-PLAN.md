# Quick Task 260602-l4c — Fix dark-mode white backgrounds in Phase 11 Research UI

## Problem

In dark mode, the Research feature's form inputs, textareas, and inactive segmented
buttons rendered with **white backgrounds**. Root cause: the Research UI used an
**undefined CSS custom property** `var(--bg, #fff)` for surfaces — `--bg` was never
defined in the editorial palette, so it always fell back to white, in every theme.
The panel correctly used the theme-aware `--paper` token (flips to #1C1A17 in dark),
but the inputs did not.

Affected (5 files): `NewResearchJobModal.tsx`, `RerunModal.tsx`, `ResearchScreen.tsx`,
`FeedbackBar.tsx`, `settings/IntegrationsSection.tsx`.

## Fix

1. **globals.css base `:root`** — add two tokens:
   - `--bg: var(--paper)` — theme-aware surface that tracks `--paper` (white in light, #1C1A17 in dark). Fixes every `var(--bg…)` *surface* usage with no per-file edit.
   - `--on-gold: #FFFFFF` — fixed white, for text sitting on gold.
2. **5 files** — leave `var(--bg…)` *surface-background* usages as-is (now theme-aware); swap the **9 sites where `var(--bg…)` is text on a `var(--gold)` background** to `var(--on-gold, #fff)` so gold buttons/chips keep crisp white labels in BOTH themes (user decision: "keep white in both themes").

## Acceptance
- Dark mode: Research modal inputs/textareas/inactive buttons render dark, not white.
- Gold buttons (Start Research, Save key, Re-run, Approve, active toggle) keep white text in both light and dark.
- No remaining `var(--bg…)` used as on-gold text; surfaces resolve via `--bg`/`--paper`.

## Scope note
`globals.css` carried ~45 lines of pre-existing dark-mode WIP; this commit bundles it
(same feature). Other unrelated WIP files left untouched.
