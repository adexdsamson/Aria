<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| voice transcript → intent router | Raw STT text is untrusted; used only for keyword classification — never passed as a prompt suffix to the write operation (D-06 anti-pattern) |
| router → person-resolver | Name tokens from transcript fed to person-resolver; resolver reads DB contacts — no injection risk since DB is local |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-06 | Tampering | Prompt injection via transcript → write | mitigate | Router uses transcript ONLY for keyword pre-filter + domain extraction (generateObject structured output). The confirm read-back reads from the persisted ApprovalRow, not from the transcript. Pitfall 5 (raw transcript in read-back) is structurally avoided by buildReadBackText reading only from row fields. |
| T-17-07 | Tampering | Staging approval_path='voice-explicit' at insert time | mitigate | insertApproval defaults to approval_path='explicit' — router MUST NOT pass 'voice-explicit'. voiceConfirm stamps it on confirm transition. Verified by Pitfall 8 in RESEARCH. |
| T-17-08 | Elevation of Privilege | Ambiguous person stageing wrong recipient | mitigate | Person-resolver disambiguation check (D-08) runs BEFORE insertApproval. If ambiguous, router returns { kind: 'ambiguous' } and does NOT stage a row — the confirmation flow aborts. |
</threat_model>
