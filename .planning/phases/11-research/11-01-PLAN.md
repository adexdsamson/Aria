<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Brave/Exa API response -> ResearchService | External JSON responses from third-party APIs; not trusted |
| Jina Reader response -> ResearchService | External markdown content from arbitrary URLs; treated as data context, not instructions |
| User supplement URLs -> Jina Reader | User-supplied URLs proxied through Jina (SSRF mitigation: Aria never fetches user URLs directly) |
| IPC renderer input -> research handlers | Renderer input validated by Zod at IPC layer; never trusted raw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | SearchProviderService -- API keys | mitigate | Keys read only via `getProviderTokens`; never stored in SQLite; never returned to renderer |
| T-11-02 | Tampering | ResearchService -- prompt injection via Jina content | mitigate | Jina page content passed as `<document>` XML-wrapped data context in generateObject prompt, not as instructions |
| T-11-03 | Elevation | IPC research handlers -- missing assertEntitled | mitigate | assertEntitled called at IPC layer for RESEARCH_JOB_CREATE and RESEARCH_JOB_RUN; static-grep ratchet whitelists research.ts (GATED_SITES update in Plan 02 Task 1) |
| T-11-04 | Elevation | SSRF via user supplement URLs | accept | Supplement URLs sent to Jina Reader (`https://r.jina.ai/<url>`); Jina is the actual fetcher; Aria makes no direct request to user-supplied origins |
| T-11-05 | Spoofing | LLM synthesis hallucination in report | mitigate | confidenceScore field surfaced to user; all source URLs preserved for independent verification |
| T-11-06 | Denial of Service | Runaway cron tasks when many jobs scheduled | mitigate | Each per-job cron key includes jobId; cancelResearchJobSchedule removes from cronRegistry on delete |
</threat_model>
