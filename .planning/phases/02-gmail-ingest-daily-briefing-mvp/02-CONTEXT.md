# Phase 2: Gmail Ingest + Daily Briefing MVP — Context

**Gathered:** 2026-05-16
**Status:** Ready for planning
**Mode:** mvp

<domain>
## Phase Boundary

First useful slice of Aria. Connect a Gmail account (read-only) and a Google Calendar
(read-only), ingest both incrementally, and produce + display the daily briefing at a
user-configurable local time (default 7am). Briefing covers three sections — today's
calendar, priority unread email, external news — each capped at top 3 items with a
one-line "why this mattered" rationale per item.

**In scope:**
- Gmail OAuth (loopback IP flow) + read-only ingest (historyId change tokens, rate-
  limited 7-day backfill on first connect, 5-minute polling)
- Google Calendar OAuth + read-only ingest (delta tokens, 15-minute polling + force-
  refresh on briefing trigger)
- Briefing engine v1: 3+3+3 sectioned doc, terse-executive tone, generated by one
  frontier LLM call routed through Phase 1's router
- News section v1: HN top stories + RSS aggregator + one Nigeria (NG) country bundle
  of curated gov/finance feeds, picked at onboarding and editable in Settings
- Briefing UI: sectioned doc layout, item rows with title + 1-line rationale, dismissible
  news items, no auto-action
- node-cron daily trigger with sleep/wake coalescing (XCUT-01) via existing
  src/main/lifecycle/powerMonitor.ts
- Status panel updates in Settings (sync state, queue depth, last error per integration)
  per XCUT-06
- Time-zone correctness across Gmail/Calendar/cron (XCUT-07)
- Expired-OAuth-token re-auth banner (EMAIL-07)

**Out of scope (deferred to later phases per ROADMAP):**
- Email priority classifier proper (Plan 3-3 sensitivity router)
- gmail.send / approval queue / drafting (Phase 3)
- Calendar write capability + smart-scheduling (Phase 4)
- Outlook/M365 (Phase 5)
- Insights derived from user's own data (Phase 8)
- Briefing feedback "more like / skip" (BRIEF-05 — Phase 8)
- News topic configuration UI beyond country/sector picker (BRIEF-04 — Phase 8)

<decisions>
## Implementation Decisions

### Briefing layout & content density
- **Format:** sectioned doc — one scrollable page with H2 sections in order:
  §Today's Calendar, §Priority Email, §News. Each item is a row with title + 1-line
  rationale + link/affordance back to source (Gmail message ID, calendar event ID,
  news URL).
- **Tone:** terse executive. Short, declarative, no filler. Example shapes:
  "Quarterly board call — you owe a slide." / "NYT: Fed paused. Affects M&A pipeline."
- **Length budget:** hard cap of top 3 per section. Forces the briefing layer to
  actually prioritize. Fits a 14" laptop viewport without scrolling.
- **MVP priority email definition** (Claude's discretion — flagged for replacement
  in Plan 3-3): unread AND `IMPORTANT` label on Gmail's side AND age ≤ 24 hours. No
  custom classifier in Phase 2.

### External news source(s)
- **Sources shipping in Phase 2:**
  1. **Hacker News top stories** — free API, no auth, no rate limits in practice.
  2. **Generic RSS aggregator** — user-pasteable feed URLs in Settings.
  3. **NG country bundle** — fixture of 3–5 Nigerian government and finance RSS feeds
     (CBN, NCC, etc.) seeded at build time. Proves the country/sector pattern; more
     country bundles ship in a dedicated later phase.
- **Picker UX:** onboarding step (new wizard page after mnemonic confirm) where user
  picks home country and 2–3 sectors of interest (gov / finance / tech / energy /
  etc.). Editable later in Settings → Briefing → News Sources.
- **Ranking:** hybrid. Source provides its top-N (HN front page / RSS feed entries
  by published date). One frontier LLM call per morning ranks the candidate pool
  against user context (job title from PROJECT.md persona + recent calendar topics
  from the last 7 days) and returns the top 3.
- **Rationale source:** same LLM call returns `{item_id, why}` pairs via
  `generateObject` + Zod schema. No per-item LLM call. Honors the Phase 1 router
  decision logging (every call writes a routing_log row).

### Ingest cadence + first-connect backfill
- **Gmail polling:** every **5 minutes** using `users.history.list` with stored
  `historyId`. Matches ROADMAP SC1 ("new mail appears in Aria within 5 minutes").
- **Gmail first-connect backfill:** **last 7 days** via `users.messages.list?q=newer_than:7d`,
  rate-limited via Phase 1's p-queue. Older messages stay untouched on Gmail.
- **Calendar polling:** every **15 minutes** using sync tokens + force-refresh on
  briefing-trigger fire. Sleep/wake coalesced via existing powerMonitor; missed
  briefings on wake are deferred to the next scheduled time (not back-fired).
- **Rate limits:** all Google API calls go through a per-integration p-queue
  (concurrency = 1 to keep Phase 1's `src/main/lifecycle/scheduler.ts` shape).

### Carried forward from Phase 1 (locked, no re-decision needed)
- Stack: Electron 41.6.1 pinned, AI SDK 6 + ollama-ai-provider-v2 + Anthropic provider,
  better-sqlite3-multiple-ciphers @ patched 12.9.0, safeStorage for OAuth tokens
- Briefing trigger: node-cron in main process, default 7am local; user-configurable
  via Settings (BRIEF-01)
- Graceful degrade (BRIEF-06): if a source fails, render its section with an inline
  warning ("Gmail unreachable — last sync 23m ago") and keep the rest of the
  briefing usable
- Expired token UX (EMAIL-07): banner inside the affected section, not a modal; other
  sections unaffected
- Status panel (XCUT-06): extend existing Settings StatusPanel with rows for Gmail
  and Calendar (sync state, queue depth, last error)
- Time zones (XCUT-07): store everything in UTC in SQLCipher; render in user's local
  TZ via `Intl.DateTimeFormat`. Cron schedule resolves against local TZ.

<specifics>
## Specific Ideas

- News source country bundle MVP: Nigeria. Bundle should ship as a JSON fixture
  under `src/main/news/bundles/ng.json` so the file IS the curated list. Pattern is
  generalized later but not now.
- Country/sector picker UI is the only new onboarding step in Phase 2 — appears after
  the Phase 1 mnemonic-confirm screen on a fresh install; appears in Settings on
  upgrade.
- The 7am briefing call is ONE frontier LLM call per day (router will likely choose
  Anthropic given non-sensitive nature; classifier treats news ranking as
  non-sensitive). Cost-bounded: ≤ ~$0.10/day at current Claude pricing.
- Calendar "today" is the user's local TZ day, not UTC day.
- News items are dismissible per briefing day, not permanently — dismissal is a row
  in a `briefing_item_dismissed` table keyed by (date, item_url_hash).

</specifics>

  ships 7-day.

  picker. Phase 2 ships hard-coded sector list per country bundle; richer
