<interfaces>
The four screens have distinct data flows. Read each existing screen file
first to extract the props/state/hooks it depends on, then replace the JSX
+ inline-style + Tailwind class output with editorial equivalents.

Pattern to follow consistently:
- Top section: `<LabelRule label="…" align="left" />` or a screen-level h2 in Playfair.
- Each grouped block: `<Card>` (`.card .card-accent-top` when the block is a hero, plain `.card` otherwise).
- Within a card, sub-rows use `.label-rule` headers when they have a labeled boundary; otherwise mono uppercase eyebrow + Playfair sub-title.
- Body text: Source Sans 3, 14-15px, line-height 1.5-1.6.
- Metadata: IBM Plex Mono 10-11px, 0.18em tracking, uppercase, gray.
- Buttons: editorial Button variants (primary=gold, outline=ink-border, ghost).
- Status: StatusDot kinds (ok/warn/err/idle) — no other status colors.
</interfaces>
