<objective>
Implement the local embedding pipeline and incremental index worker so Aria can build, refresh, and version a local RAG index without sending embedding input to frontier services.
</objective>

<context>
Read:
- `.planning/phases/07-rag-q-a/07-01-SUMMARY.md`
- `.planning/phases/07-rag-q-a/07-SPIKE-CHUNKING.md`
- `src/main/rag/chunk-types.ts`
- `src/main/rag/source-harvesters.ts`
- Existing queue, cron, and power-monitor patterns from earlier phases
- Settings page integration patterns
</context>

<constraints>
- All embeddings must go through local Ollama using `nomic-embed-text` or the configured local embedding model.
- Do not block the renderer while indexing or rebuilding.
- Queries must filter by active `model_id`; old vector rows may exist during rebuild but must not be silently reused after an active model change.
- Source deletes must remove chunks, embeddings, and FTS rows.
- If sqlite-vec cannot load in the current Windows/SQLCipher environment, implement a tested local fallback interface and document the limitation.
</constraints>
