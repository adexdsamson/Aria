<objective>
Build the first-run model-download manager: a main-process service wrapping node-downloader-helper
to fetch the Whisper large-v3-turbo q5_0 GGML (~574 MB) from the ggml-org/whisper.cpp Hugging Face
repo into userData, with HTTP Range resume, size disclosure before start, progress events pushed to
the renderer, powerMonitor pause-on-sleep, and a model-readiness flip on completion (D-05 through D-09).

Plan 15-05; IPC wiring is Plan 15-06. This plan is the engine, unit-tested with a mocked DownloadHelper.

package.json — project memory: lockfile CI drift).

|----------|-------------|

| download → filesystem | The model file lands in userData (SQLCipher does not cover loose files) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-08 | Tampering | model download integrity (supply chain) | mitigate | download from the pinned HF ggml-org/whisper.cpp URL over HTTPS; validate final file size against disclosedSize() before flipping readiness — a size mismatch keeps readiness not-ready (a truncated/MITM'd file fails the gate). (Full SHA256 checksum verification is a recommended hardening if HF publishes one; size+HTTPS is the L1 floor here.) |
| T-15-09 | Information Disclosure | model fetch egress | accept | fetch is to the HF CDN only; no audio or user data is sent — the request body is a bare GET; covered by the no-cloud-audio ratchet (model URL ≠ audio endpoint) |
| T-15-10 | Denial of Service | interrupted download | mitigate | resumeIfFileExists + resumeOnIncomplete (Range) recover from interruption; powerMonitor pauses on sleep to avoid a cron-storm-style retry burst |
</threat_model>
