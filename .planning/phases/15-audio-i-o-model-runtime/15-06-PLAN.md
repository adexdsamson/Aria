<objective>
Build the renderer TTS path and the voice session state machine — the two pieces that prove the
half-duplex gate (the primary VOICE-07/SC3 deliverable). useKokoroPlayer stands up the REAL Kokoro-82M
engine in the renderer (webgpu→wasm fallback) triggered by a minimal utterance (D-18 — a real engine,
not a placeholder beep, so TTS packaging and the RAM ceiling are proven). useVoiceSession is the Zustand
state machine (idle→listening→processing→speaking→idle, with the 'speaking' state present per D-17) that
owns the micGated half-duplex gate: micGated=true on turn-start AND for the entire TTS playback duration,
PTT start blocked while speaking, gate released after an ~800ms cooldown (D-13).

Purpose: VOICE-07 ("mic state always visible + gated during playback so Aria never transcribes its own
TTS") and SC3 (verified on laptop speakers) rest on this gate. D-18's real Kokoro path is what lets SC3
be verified against the real acoustic echo path and what makes the RAM-ceiling proof (Plan 15-07) honest.
Output: the Kokoro player hook, the session store, and the half-duplex unit proof.

This plan owns the half-duplex logic and the store; the UI components that render the state (StatusDot,
HUDBand, PTT button) and the IPC bridge already exist (Plan 15-01 contracts) — the visual surface is
Plan 15-07.
</objective>
