<objective>
Build the STT sidecar manager: a persistent Node-process wrapper that owns the whisper.cpp CLI
binary lifecycle and the VAD-endpoint → temp-WAV → `whisper-cli --output-json` → JSON-lines
transcript protocol (D-01/D-03). This is the runtime that makes SC2's "no NODE_MODULE_VERSION
ABI crash on packaged launch" TRUE by construction — a pure CLI binary cannot throw that error.

Purpose: VOICE-04 (on-device by default) and the ABI-safety guarantee both rest on this design.
The sidecar is crash-isolated from the app process; a whisper.cpp segfault never takes down Aria.
Output: the sidecar manager, a WAV writer, and a static ratchet proving the STT path ships no
`.node` addon.

This plan does NOT procure the macOS binary (Plan 15-09 packaging) and does NOT wire IPC
(Plan 15-05). It builds the manager against a configurable binary path with a unit-test seam that
mocks the child process — no real binary required to test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main → child process (whisper-cli) | The sidecar spawns a separate process; a segfault is isolated from the app |
| audio data → disk (temp WAV) | PCM is briefly materialized as a temp WAV the CLI reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Information Disclosure | temp WAV persistence | mitigate | temp WAV written to os.tmpdir() and deleted in try/finally on success AND crash/exit (§Pitfall 4); never written to userData |
| T-15-05 | Denial of Service | zombie child processes | mitigate | dispose()/crash handler reaps the child; powerMonitor parks on suspend, respawns on resume (D-03) |
| T-15-06 | Tampering | sidecar binary integrity | transfer | binary code-signing/notarization is Plan 15-09 packaging (electron-builder mac.binaries + Windows OV); manager resolves only from process.resourcesPath in packaged builds |
| T-15-07 | Elevation of Privilege | voice → write chokepoints | mitigate | file lives under src/main/voice/**; voice-routes-through-staging ratchet bans direct chokepoint calls |
</threat_model>
