# Phase 15 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase delivers the mic-state surface — a persistent Topbar StatusDot + a transient
> in-flow VoiceHUDBand — plus a first-run voice-model download flow and a "voice unavailable
> until ready" disabled PTT affordance. Hold it to Aria's editorial bar: calm, ivory/ink/gold,
> Playfair Display + IBM Plex Mono. NOT a flashy consumer-assistant orb.

### Accent reserved for

Gold (`var(--gold)`) is reserved ONLY for:
1. Listening state in StatusDot (slow pulse animation)
2. Processing state in StatusDot (spinner arc overlay)
3. Whisper model download progress bar fill
4. PTT button active/held ring highlight
5. "Set up voice" step title eyebrow label (uppercase mono)
6. Download size disclosure text label prefix ("DOWNLOAD SIZE")

Gold must NOT be used for: generic UI buttons, all-caps eyebrows in contexts unrelated to voice state, or the disabled-PTT affordance.

### State-to-color encoding (D-14)

| Voice State | StatusDot color | Label (mono, uppercase) |
|-------------|-----------------|-------------------------|
| idle | `var(--gray-faint)` — no pulse | (no label in HUD — collapsed) |
| listening | `var(--gold)` — slow pulse (2s ease-in-out) | `LISTENING` |
| processing | `var(--gold)` + spinner arc | `PROCESSING` |
| speaking | `var(--moss)` — static | `SPEAKING` |
| muted-during-playback | `var(--gray-faint)` + struck-mic icon | `MUTED` |
| error | `var(--rose)` — static | `ERROR` |

Pulse animation: `@keyframes voice-pulse { 0%,100%{opacity:1} 50%{opacity:0.45} }` at 2s ease-in-out. Suppressed when `prefers-reduced-motion: reduce` is set — dot stays static gold fill instead.

## Component Inventory

### 1. VoiceStatusDot (new) — `src/renderer/features/voice/VoiceStatusDot.tsx`

**What it is:** A thin wrapper around the existing `StatusDot` editorial primitive that maps `VoiceState` → `StatusDotKind`. Placed between the bell and `AvatarMenu` in the Topbar right cluster.

**Topbar insertion slot:** Between `aria-topbar-bell` and `AvatarMenu`. The existing right-cluster order is: `⌘K button → bell → AvatarMenu`. New order: `⌘K button → bell → VoiceStatusDot → AvatarMenu`.

**Props:**
```ts
interface VoiceStatusDotProps {
  state: VoiceState; // 'idle' | 'listening' | 'processing' | 'speaking' | 'muted' | 'error'
}
```

**State → kind mapping:**
- `idle` → `kind='idle'` (gray-faint dot, no pulse)
- `listening` → `kind='warn'` (gold dot + pulse keyframe)
- `processing` → `kind='warn'` (gold dot + spinner arc overlay — see below)
- `speaking` → `kind='ok'` (moss dot, static)
- `muted` → `kind='idle'` (gray-faint dot) + inline struck-mic SVG icon (12×12) positioned beside it
- `error` → `kind='err'` (rose dot, static)

**Processing spinner arc:** Rendered as a 10×10 SVG `<circle>` arc (stroke-dasharray) inside a `position: relative` wrapper around the StatusDot. Stroke color: `var(--gold)`. Rotation: CSS `@keyframes spin` at 700ms linear infinite. Suppressed under `prefers-reduced-motion` (arc disappears; gold dot alone signals processing).

**Accessibility:** `aria-label="Microphone: {STATE}"` on the wrapper `<span>`. Updated via `aria-live="polite"` on a visually-hidden sibling — NOT on the dot itself to avoid announcing every pulse tick.

**Data-testid:** `aria-topbar-voice-dot`

### 2. VoiceHUDBand (new) — `src/renderer/features/voice/VoiceHUDBand.tsx`

**What it is:** A transient in-flow banner — same structural slot as `TrialBanner` in `App.tsx` — that expands from 0 to full when voice is active and collapses when idle. Never a floating overlay.

**Shell integration:** In `App.tsx`'s unlocked layout, mount `VoiceHUDBand` immediately before `TrialBanner` (both sit above `<Topbar />`). Rationale: VoiceHUDBand takes priority over the trial prompt in layout order when both are visible simultaneously.

**Expansion technique:** `grid-template-rows: 0fr → 1fr` (D-Claude's discretion resolved per RESEARCH.md recommendation — cleaner, no magic max-height). The inner content wrapper has `overflow: hidden`. Transition: `grid-template-rows 200ms cubic-bezier(.2,.6,.2,1)`. Under `prefers-reduced-motion`: no transition, instant toggle.

**Idle state:** `grid-template-rows: 0fr` — band renders with zero height, no layout impact. The component mounts unconditionally in the shell to avoid mount/unmount layout jank.

**Active states (listening / processing / speaking / muted / error):** `grid-template-rows: 1fr` — band expands to reveal inner content.

**Inner layout:**
```
[16px padding] [state dot 6px] [8px gap] [mono label 10px] [16px gap] [transcript text 14px, flex-1] [16px padding]
```

**Background:** `var(--ivory)`. Bottom border: `1px solid var(--rule)`. No top border (sits above Topbar which has its own bottom rule).

**Transcript rendering (honest, per RESEARCH.md constraint):**
- State `listening`: Shows `"Listening…"` with the gold StatusDot. No partial words — whisper-cli is file-based; no mid-utterance stream exists. A subtle animated ellipsis (CSS `@keyframes dots`) signals activity.
- State `processing`: Shows `"Transcribing…"` with the gold+spinner dot. Transcript field blank.
- When utterance resolves (transcript arrives): Renders the full utterance text in `var(--ink)`, weight 400, 14px Source Sans 3. The state label changes to the next appropriate state. Previous transcript is replaced (not appended) per turn.
- State `speaking`: Shows `"Speaking…"` with the moss dot. Transcript field shows the echo text being spoken.
- State `muted`: Shows `"Mic muted during playback"` in `var(--gray)`, italic, 13px.
- State `error`: Shows error message text in `var(--rose)`, 13px.

**ARIA contract:** `role="status"`, `aria-live="polite"`, `aria-atomic="false"`. The label + transcript are in a single live region. `aria-atomic="false"` so screen readers announce the transcript text as it arrives, not the full buffer each update.

**Data-testid:** `voice-hud-band`, `voice-hud-state-label`, `voice-hud-transcript`

### 3. VoicePTTButton (new) — `src/renderer/features/voice/VoicePTTButton.tsx`

**What it is:** The push-to-talk control. Supports BOTH hold-to-talk (keydown/keyup) AND click-toggle affordances on the same control. Both affordances must be discoverable — described below.

**Visual form:** A 44×44px circular button. Not a standard rectangular `.btn`. Uses `border-radius: 50%` with a centered mic SVG icon (inline, 18×18).

**States:**

| Voice State | Visual |
|-------------|--------|
| voice unavailable | `opacity: 0.4`, `cursor: not-allowed`, `pointer-events: none`. Tooltip on hover: "Set up voice first". Routes to download flow on click (see VoiceModelDownload). |
| idle (available) | `background: var(--paper)`, `border: 1.5px solid var(--rule)`, mic icon color `var(--gray)`. |
| hold-active (key held) | `background: var(--ivory-deep)`, `border: 1.5px solid var(--gold)`, `box-shadow: 0 0 0 4px rgba(184,134,11,0.15)`. Mic icon color `var(--gold)`. Gold ring pulses at 1.5s. |
| toggle-active (recording) | Same as hold-active. Click again to stop. |
| muted (TTS playing) | `opacity: 0.5`, `cursor: not-allowed`. Mic icon has struck-line overlay. Tooltip: "Aria is speaking". |

**Discoverability of both hold + toggle:** A `<kbd>` hint label is rendered below the button at 10px mono gray: `"Hold · Click to toggle"`. This surfaces both affordances at a glance without a tooltip dependency.

**Keyboard binding:** `Space` key hold-to-talk (consistent with broadcast/PTT conventions). Displayed in the hint label as `"Space · Click to toggle"`.

**Placement:** Aria's current layout does not have a persistent action dock. The PTT button is placed in the `VoiceHUDBand` inner-right slot — visible when the HUD is expanded — AND as a persistent affordance in the Topbar right cluster at `idle` (collapsed) state. The Topbar slot is a compact 28×28px icon button (not the full 44px circle) that expands the HUD on activation.

**Accessibility:** `aria-label="Push to talk — hold Space or click to toggle"`. When muted: `aria-disabled="true"`. When unavailable: `aria-describedby="voice-unavailable-hint"`.

**Data-testid:** `voice-ptt-button`

### 5. VoiceModelDownload (new) — `src/renderer/features/voice/VoiceModelDownload.tsx`

**What it is:** Used in two contexts: (a) as the "Set up voice" step inside `OnboardingWizard`, and (b) as a standalone lazy first-PTT modal for users who skipped onboarding.

#### 5a. Onboarding step variant

Rendered as a new step card inside the existing `OnboardingWizard` step flow. Follows the same card structure (`Card` editorial primitive) and wizard chrome.

**Step position:** Appended after the existing `password` step — the final step before `sealing`. The new step type is `'voice'`.

**Content layout:**
1. Gold mono eyebrow (10px): `"VOICE ASSISTANT"`
2. Playfair Display heading (20px, weight 500): `"Set up voice for Aria"`
3. Source Sans body (14px, `var(--gray)`): `"Aria can transcribe your voice locally — no audio leaves your machine. Download the speech model to get started."`
4. **Size disclosure block** (before any download starts — mandatory per D-07):
   - IBM Plex Mono, 11px, `var(--gray)`: `"DOWNLOAD SIZE"`
   - Source Sans 3, 14px, `var(--ink)`: `"Whisper large-v3-turbo — 574 MB"`
   - Source Sans 3, 12px, `var(--gray-soft)`: `"Stored in your app data folder. One-time download."`
5. Progress bar (appears after download starts):
   - Container: `height: 4px`, `background: var(--rule)`, `border-radius: 2px`
   - Fill: `background: var(--gold)`, animated width. Under `prefers-reduced-motion`: no animation, static fill based on current percent.
   - Below bar: `"NN% · X.X MB / 574 MB"` — IBM Plex Mono 11px, `var(--gray)`, tabular-nums.
   - Pause/resume link: `"Pause"` / `"Resume"` — ghost text link style, `var(--gold)`, 12px.
6. CTA button row:
   - Primary: `"Download now"` (`.btn .btn-primary`, gold)
   - Secondary: `"Set up later"` (`.btn .btn-ghost`, `var(--gray)`) — skips the step, sets model-readiness to pending. The user will see the lazy modal on first PTT.

**During download:** "Download now" button changes to `"Downloading…"` with `disabled` state and spinner. Pause link appears. "Set up later" becomes "Skip" in ghost style.

**Download complete state:**
- Progress bar replaces with a checkmark row: moss-colored SVG check + `"Voice ready"` in `var(--moss)`, 14px.
- CTA changes to `"Continue →"` (primary) to advance to the next onboarding step (`sealing`).

#### 5b. Lazy first-PTT modal variant

A `Modal` editorial primitive (`src/renderer/components/editorial/Modal.tsx`) at `size="md"` (existing `xl` size added in Phase 9 is too large; use `md`).

**Trigger:** When user activates PTT (hold Space or click) and `voice_model_ready = 0`.

**Modal content:** Same disclosure + progress content as 5a, adapted for modal chrome. Heading: `"Download the voice model"`. Body: same size disclosure + download flow.

**Modal close:** `"Set up later"` button dismisses modal and blocks PTT for the session. The StatusDot shows `error` state with tooltip `"Voice model not downloaded"` while the session is blocked.

**Data-testids:** `voice-model-download-step` (onboarding), `voice-model-download-modal` (lazy), `voice-download-cta`, `voice-download-skip`, `voice-download-progress-bar`, `voice-download-size-disclosure`

### 6. "Voice unavailable until ready" disabled PTT affordance (D-08)

When `voice_model_ready = 0` and the user is past onboarding:

- Topbar PTT icon: `opacity: 0.4`, `cursor: default`, renders but is non-interactive.
- VoiceHUDBand: Never expands — no voice session can start.
- On click of the disabled PTT icon: Opens the lazy first-PTT modal (VoiceModelDownload 5b).
- Aria label on disabled state: `aria-describedby="voice-unavailable-hint"` pointing to a visually-hidden `<span id="voice-unavailable-hint">Set up voice to use this feature</span>`.

This mirrors the TrialBanner entitlement-gate disabled pattern — same idea: a visible but non-operable affordance that routes to a setup/unlock flow.

## Interaction Contract

### Push-to-Talk (D-10)

| Action | Trigger | Result |
|--------|---------|--------|
| Start hold | `keydown Space` (focused window) | Enter `listening` state. StatusDot → gold + pulse. HUD band expands. "Listening…" shown. |
| Stop hold | `keyup Space` | End turn. Enter `processing` state. StatusDot → gold + spinner. HUD → "Transcribing…". |
| Start toggle | Click PTT button | Same as start hold. |
| Stop toggle | Click PTT button again | Same as stop hold. |
| Transcript arrives | `VOICE_TRANSCRIPT_DELTA` IPC push | HUD renders full utterance text. State returns to `idle` or `speaking` per pipeline. |
| TTS playback starts | `VOICE_TTS_PLAYING` IPC push | State → `speaking`. `micGated = true`. PTT button → muted/disabled. |
| TTS playback ends | `VOICE_TTS_DONE` IPC push | State → `idle`. `micGated = false` after 800ms cooldown. |
| Permission denied | Error from `getUserMedia` | State → `error`. ToastHost notification. HUD shows error message. |
| Device change | `devicechange` event | Seamless re-acquisition; if fails → `error` state via ToastHost. |

### HUD sequence per utterance (honest model, per RESEARCH.md)

```
idle → [user holds PTT / keydown]
    → listening: HUD expands, shows "Listening…" + animated ellipsis + gold dot
    → [user releases PTT / keyup, or VAD fires onSpeechEnd on toggle]
    → processing: HUD shows "Transcribing…" + gold+spinner dot (whisper-cli running)
    → [whisper-cli returns JSON transcript]
    → transcript rendered: HUD shows utterance text in ink, dot returns to idle
    → [minimal Kokoro TTS playback triggered]
    → speaking: HUD shows "Speaking…" + echo text + moss dot, mic gated
    → [TTS finishes]
    → idle: HUD collapses after 2s delay (so transcript is readable before collapse)
```

No word-by-word partials. "Live transcription" (SC1/VOICE-01) means the transcript appears per-utterance after endpointing. The HUD's `listening` state must read as active and purposeful — animated ellipsis + "Listening…" label — not blank.

### Reduced Motion (D-16)

Under `@media (prefers-reduced-motion: reduce)`:
- All CSS keyframe animations disabled (`animation: none`).
- HUD expansion: `transition: none` — instant show/hide.
- Gold pulse on listening dot: replaced by solid gold fill.
- Processing spinner arc: hidden. Gold dot alone signals processing.
- Progress bar fill: no width transition — jumps to current value.
- Animated ellipsis: replaced by static `"…"`.

State information remains fully accessible via: color (`var(--gold)` / `var(--moss)` / `var(--rose)` / `var(--gray-faint)`), IBM Plex Mono uppercase label, and `aria-live` announcement.

## Accessibility Contract

- `VoiceHUDBand`: `role="status"`, `aria-live="polite"`, `aria-atomic="false"`.
- `VoiceStatusDot` wrapper: `aria-label="Microphone: {STATE}"`. Sibling visually-hidden live region announces state changes to screen readers without coupling to animation ticks.
- `VoicePTTButton`: `aria-label="Push to talk — hold Space or click to toggle"`. `aria-disabled="true"` when muted. `aria-describedby="voice-unavailable-hint"` when model not ready.
- Download progress: `role="progressbar"`, `aria-valuenow={percent}`, `aria-valuemin={0}`, `aria-valuemax={100}`, `aria-label="Voice model download progress"`.
- Error toasts route through existing `ToastHost` (Phase 9 editorial pattern) — no custom z-index layer introduced.
- Focus management: When the lazy first-PTT modal opens, focus moves to the modal heading. When closed, focus returns to the PTT button.
- Keyboard: PTT Space hold-to-talk works only when `document.activeElement` is within the main app shell (not inside an input/textarea). Guard: `if (e.target instanceof HTMLInputElement || e.target instanceof HTMLTextAreaElement) return;`.
