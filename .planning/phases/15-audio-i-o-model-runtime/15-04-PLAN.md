<objective>
Build the renderer audio-capture layer: getUserMedia → inline-Blob-URL AudioWorklet → 16 kHz mono
PCM, with device hot-swap and permission-denied handling (D-19/D-20). This is the front of the
VOICE-01 pipeline — it produces the transferable ArrayBuffer PCM frames that Plan 15-05 streams to
the sidecar, and it is the SC5 surface (device hot-swap + 16 kHz resample + permission-denied as an
actionable error).

Purpose: there is no existing getUserMedia/AudioWorklet code in Aria (no analog) — this plan is the
canonical capture implementation against RESEARCH §Pattern 2/§Pattern 5 excerpts. It depends on the
Plan 15-01 CSP `blob:` fix; without it the worklet would silently fail to register in the packaged build.
Output: the worklet source + Blob-URL loader and the useMicCapture hook with hot-swap + error routing.

This plan does NOT own VAD turn-detection wiring (the VAD dual-role config lives with the PTT button,
Plan 15-05) and does NOT wire IPC (Plan 15-05). It exposes a clean onPcmFrame(ArrayBuffer) callback and
an error channel the session store consumes.
</objective>
