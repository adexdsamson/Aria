## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build pipeline → shipped binary | the sidecar binary enters the app bundle; signing/notarization is the integrity gate |
| packaged app → OS (Gatekeeper) | an unsigned binary inside a notarized .app is quarantined and fails to spawn |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-27 | Tampering | sidecar binary integrity (supply chain) | mitigate | macOS binary in build.mac.binaries → electron-builder Developer-ID code-signs it (§Pitfall 2); Windows OV signing of the app covers whisper-cli.exe; macOS notarization covers Resources/; the static guard locks the config |
| T-15-28 | Tampering | binary provenance | mitigate | Option A CI build (cmake) gives known provenance + otool -L self-contained verification; Option B third-party is flagged temporary-only with unverified provenance |
| T-15-29 | Denial of Service | Gatekeeper quarantine on clean install | mitigate | the mac.binaries signing entry + notarization prevent the quarantine that would fail spawn() on a clean machine; the SC2 E2E runs on the packaged build, not the dev machine where quarantine is cleared |
| T-15-30 | Information Disclosure | RAM-pressure swap of audio buffers | accept | RAM-ceiling proof confirms headroom on the target 16 GB machine; swapped PCM is transient + local; no network egress |
</threat_model>
