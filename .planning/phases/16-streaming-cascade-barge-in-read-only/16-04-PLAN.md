<objective>
Wire the full streaming cascade end-to-end: the main-process VoiceSessionManager orchestrating
TtsSegmenter + streamVoiceAnswer + VOICE_TTS_CHUNK push, the renderer VoiceHUDBand gaining
transport controls, and BriefingScreen gaining briefing read-aloud.

chunk via Kokoro; (b) press PTT during playback to barge in and be heard; (c) press the
briefing speak button to hear the daily briefing section-by-section with pause/skip/speed.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| STT transcript → VOICE_FEED_ANSWER IPC | Question text crosses renderer→main; length-capped at 4096 in streamVoiceAnswer |
| main streamText → renderer VOICE_TTS_CHUNK | LLM output crosses main→renderer as plain text; no PII since LOCAL-only |
| BriefingPayload → TTS speech | Stored briefing content (pre-generated) read aloud; no new network path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-10 | Tampering | VOICE_FEED_ANSWER question payload | mitigate | 4096-char cap in streamVoiceAnswer; question treated as data in prompt template (no eval) |
