## 1. JWT / Ed25519 Library Choice

**Pick: `jose` (v5.x) â€” both in Aria desktop client and Cloudflare Worker.**

Rationale:
- `jose` ships Ed25519 (EdDSA / OKP) verification with **zero native deps**, ~25 KB ESM after tree-shaking. Pure WebCrypto under the hood â€” same code path in Node 20 (Electron main) and the Worker runtime.
- `@noble/ed25519` + custom JWT framing is smaller (~6 KB) but every byte of JWT plumbing (b64url, header parsing, claim validation, exp/iat enforcement, kid rotation) becomes our problem. Cost > benefit for a single phase.
- `jsonwebtoken` is rejected: pulls in CommonJS crypto, no native Ed25519 verify in older versions, dead-on-arrival in Workers.

Worker-side signing uses `jose.SignJWT` with a `Ed25519` `KeyLike` materialized from a raw 32-byte seed stored in the Worker's secret store (`wrangler secret put ENT_SIGNING_SEED`).

Aria-side verification uses `jose.jwtVerify` against a single hard-coded public key embedded in the app (no remote JWKS for v1 â€” rotation is "ship an Aria update"; document this in plan 08.1-02 README).

## 3. Customer Portal Session Creation

**Pick: Route through the license server, not direct from app.**

Rationale:
- Direct-from-app requires shipping a Stripe restricted key with `portal_session:write` scope. Even restricted, a leaked key from a customer machine could be abused to mint portal sessions for OTHER customers. This is the same reason webhook secrets stay server-side.
- License server already has the `customer_id` â†” `license_key` mapping in D1 (it minted the key on `checkout.session.completed`). Adding `POST /v1/portal/session { license_key, install_id } â†’ { url }` is ~20 LoC and inherits the existing JWT auth path.
- Latency cost is negligible â€” portal sessions are user-initiated (click "Manage subscription") with an existing loading affordance.

Sequence: user clicks "Manage" in Aria â†’ IPC calls Worker `/v1/portal/session` with the current entitlement JWT â†’ Worker validates JWT + maps to `stripe_customer_id` â†’ calls `stripe.billingPortal.sessions.create({ customer })` â†’ returns short-lived URL â†’ Aria opens with `shell.openExternal(url)`.

## 4. License Server Endpoint Contracts (final)

All endpoints `application/json` over HTTPS. Worker is stateless; D1 is the only store. Each endpoint returns either `{ ok: true, ... }` or `{ ok: false, error: { code, message } }` with appropriate HTTP status. Codes are documented strings (`already-bound`, `key-not-found`, `key-revoked`, `install-cap-exceeded`, `invalid-signature`, `clock-skew`, `rate-limited`).

### `POST /v1/trial/start`
- Body: `{ install_id: string (uuidv4) }`
- Returns: `{ ok: true, trial_started_at: ISO8601, jwt: string }`
- JWT claims: `{ sub: "trial:" + install_id, iat, exp: iat + 7d, tier: "trial", install_id, features: [], trial_started_at, trial_expires_at }`
- D1 row: `INSERT INTO trials(install_id, trial_started_at, ...) ON CONFLICT(install_id) DO NOTHING RETURNING trial_started_at`. Idempotent â€” replay returns the original timestamp so the user can't "reset trial" by reinstalling on the same machine (within the install_id key space â€” see Â§8 threat model).
- Rate limit: 10 requests / IP / hour via Workers' rate-limiting binding.

### `POST /v1/license/activate`
- Body: `{ license_key: string, install_id: string }`
- Returns: `{ ok: true, jwt: string }` or `{ ok: false, error: { code: 'install-cap-exceeded' | 'key-not-found' | 'key-revoked' } }`
- Binds `install_id` to `license_key` if `< 3` bindings exist. Atomic via D1 transaction with `SELECT ... FOR UPDATE` emulation (D1 single-writer-per-DB makes this safe with naive count + insert in one batch).
- JWT claims: `{ sub: license_id, iat, exp: iat + 7d, tier: "pro", install_id, features: [], license_key_hash }`. (`license_key_hash` is sha256 â€” not the raw key â€” to limit damage if a JWT leaks.)

### `POST /v1/entitlement/refresh`
- Body: `{ jwt: string, install_id: string }`  (must match `jwt.install_id`)
- Returns: `{ ok: true, jwt: string }` (new 7d JWT) or `{ ok: false, error: { code: 'revoked' | 'subscription-canceled' | 'invalid-signature' } }`
- This is the 24h online check. Worker re-validates: subscription status in D1 (most recent webhook fact), install binding still active, key not revoked.

### `POST /v1/license/resend`
- Body: `{ stripe_customer_session_token }` (from Stripe Customer Portal magic link flow)
- Returns: `{ ok: true }` (email dispatched)
- Customer-portal-driven; no JWT required because Stripe authenticated the customer.

### `POST /v1/portal/session`
- Body: `{ jwt }`
- Returns: `{ ok: true, url: string }` â€” short-lived Stripe portal session URL.

### `POST /v1/webhooks/stripe`
- Stripe signs with `Stripe-Signature` header â€” verify with `stripe.webhooks.constructEvent(body, sig, STRIPE_WEBHOOK_SECRET)`.
- Events handled:
  - `checkout.session.completed` â†’ generate license key â†’ upsert into D1 `licenses` table â†’ email key via Resend.
  - `customer.subscription.updated` â†’ update D1 `subscription_status`.
  - `customer.subscription.deleted` â†’ set `status = 'canceled'`; future `/v1/entitlement/refresh` will return `subscription-canceled` until JWT expires (7d max), then app locks.
  - `invoice.payment_failed` â†’ record but DO NOT immediately revoke (Stripe has its own dunning); only if subscription transitions to `unpaid` after dunning does refresh return `error`.
- Idempotency via `stripe.events.id` UNIQUE constraint on D1 `webhook_events` table.

## 5. License Key Format

`ARIA-<ULID>-<checksum>` where ULID is 26 chars Crockford base32 (time-ordered, lexicographically sortable for support tooling) and checksum is the last 4 chars of `sha256(ULID + SECRET_SALT)` upper-cased. Total length 36 chars. Server-side validation rejects malformed keys without a D1 lookup (cheap DOS defense).

Example: `ARIA-01HV3K7QGS5RFRW7VYTYZ9XYZA-9F2B`

## 6. D1 Schema (license server, separate repo)

```sql
CREATE TABLE trials (
  install_id TEXT PRIMARY KEY,
  trial_started_at TEXT NOT NULL,
  ip_hash TEXT,  -- sha256 of source IP for abuse detection only
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE licenses (
  license_id TEXT PRIMARY KEY,             -- ULID
  license_key TEXT NOT NULL UNIQUE,        -- full ARIA-xxx-xxxx form
  stripe_customer_id TEXT NOT NULL,
  stripe_subscription_id TEXT NOT NULL,
  status TEXT NOT NULL,                    -- active | past_due | canceled | unpaid
  email TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_licenses_customer ON licenses(stripe_customer_id);

CREATE TABLE installs (
  license_id TEXT NOT NULL,
  install_id TEXT NOT NULL,
  bound_at TEXT NOT NULL DEFAULT (datetime('now')),
  last_seen_at TEXT,
  PRIMARY KEY (license_id, install_id),
  FOREIGN KEY (license_id) REFERENCES licenses(license_id)
);
CREATE INDEX idx_installs_install ON installs(install_id);

CREATE TABLE webhook_events (
  event_id TEXT PRIMARY KEY,               -- Stripe event id (idempotency)
  type TEXT NOT NULL,
  received_at TEXT NOT NULL DEFAULT (datetime('now')),
  status TEXT NOT NULL                     -- processed | failed | duplicate
);
```

## 8. Threat Model (for `gsd-secure-phase`)

| Threat | Mitigation |
|---|---|
| **Local clock manipulation to extend trial** | `trial_started_at` is server-signed in JWT; gate computes lock from `trial_expires_at` claim, NOT local timestamp. If `now() > trial_expires_at` per JWT claim â†’ lock, no matter what local clock says. |
| **Reinstall to reset trial** | `install_id` is a UUID generated once on first launch and stored in OS keychain via `safeStorage` (NOT just in encrypted DB â€” survives DB wipe). Reinstall on same machine returns the same `trial_started_at` from D1. Wipe-keychain-then-reinstall is a defeat we accept (cost > value at this stage; documented). |
| **License-key sharing across â‰¥4 installs** | Install binding cap of 3 enforced atomically at `/v1/license/activate`. |
| **Replayed JWT after revocation** | JWT exp â‰¤ 7d caps blast radius. `/v1/entitlement/refresh` rejects revoked licenses. Lock occurs â‰¤ 7d + 14d grace = 21d worst case after revocation. |
| **SQLCipher tampering to flip tier='pro'** | `assertEntitled()` re-verifies JWT signature on every call (not just on read into memory). Tampered row â†’ invalid signature â†’ lock. |
| **Stripe webhook spoofing** | `stripe.webhooks.constructEvent` signature verification mandatory; reject without 200. |
| **Worker secret leak (signing seed)** | Single-line rotation: deploy new public key in next Aria release, double-sign for 7d window. Documented in operations runbook (deferred until needed). |
| **DOS against `/v1/entitlement/refresh`** | Cloudflare rate-limiting binding caps per-IP and per-install_id. App-side: refresh runs once per 24h; aggressive retries on transient failures use exponential backoff with jitter. |
| **PII in license server logs** | Log `install_id` (random UUID, not user-attributable until purchase), `license_key_hash` (sha256), `email` ONLY on Stripe webhook receive. No prompt content, no app data. |

## 9. Trial Countdown / Grace UX States (functional, not visual)

Day-based banner triggers â€” computed from `trial_expires_at` claim:

| State | Trigger | UI affordance |
|---|---|---|
| `trial-active-quiet` | `days_remaining > 10` | No banner |
| `trial-active-day50` | `days_remaining == 10` | Subtle banner top of briefing: "10 days left in trial. Subscribe â†’" |
| `trial-active-day55` | `days_remaining == 5` | Stronger banner persistent across screens |
| `trial-active-day59` | `days_remaining âˆˆ [1, 2]` | Modal on app launch (dismissable), bright top banner |
| `trial-expired-grace` | `days_remaining == 0 && now < trial_expires_at + 24h` | Full-screen paywall on next write action; reads still work |
| `trial-locked` | `now >= trial_expires_at` | Full paywall blocks all gated surfaces |
| `pro-active` | `tier == 'pro' && now < jwt.exp + 14d grace from last_verified_at` | No banner |
| `pro-grace` | `tier == 'pro' && offline-grace counter < 14d` | Subtle banner: "Couldn't verify subscription. Check back online soon." |
| `pro-locked` | `tier == 'pro' && offline-grace exhausted (last_verified_at + 14d < now)` | Same paywall as `trial-locked` until next online check succeeds |
| `clock-skew-warn` | `local clock skew vs JWT iat > 7d` | Warning banner: "Check your system clock." NOT a lock state. |

State machine is a **pure function** of (current entitlement row, current time, server-side facts via last successful refresh). Lives at `src/main/entitlement/state.ts` â€” pure, testable, no I/O.

## 10. Deep Link `aria://` Registration

| OS | Mechanism | Where |
|---|---|---|
| macOS | electron-builder `protocols` field in `build.mac` (auto-fills Info.plist `CFBundleURLTypes`) | `package.json` build block |

| Linux | electron-builder `protocols` field (writes `.desktop` MimeType) | `package.json` build block |

Single `protocols` entry covers all three:
```json
"protocols": [{ "name": "Aria", "schemes": ["aria"] }]
```

Runtime: `app.setAsDefaultProtocolClient('aria')` on first launch (idempotent). Single-instance lock (already implemented at `src/main/single-instance.ts`) routes incoming `aria://activate?key=...` URLs to the running instance via `second-instance` event payload (Windows/Linux) or `open-url` event (macOS).
