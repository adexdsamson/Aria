`EntitlementProvider`:
- Loads initial state via `window.aria.entitlement.getState()` on mount
- Subscribes via `window.aria.entitlement.onStateChanged(setState)` on mount; calls the returned unsubscribe fn in the effect cleanup
- Exposes state + action wrappers (`refresh`, `activate`, `openCheckout`, `openPortal`) via context
- Shows a brief loading splash while initial state is being fetched (do NOT render children with `undefined` state — defensive against race conditions)

`useEntitlement`:
- Reads context; throws if used outside the provider (standard pattern)

Both files thoroughly typed against the shared types exported by `src/shared/ipc-contract.ts`.

Apply React useEffect best practices: no derived-state-in-effects; the subscription effect has an empty dependency array and is the SOLE source-of-truth subscription point.
