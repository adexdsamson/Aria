# Phase 08.1: Subscription & 60-Day Trial — Context

**Gathered:** 2026-05-20
**Status:** Ready for planning
**Mode:** Pre-locked decisions + batched gray-area discussion

<domain>
## Phase Boundary

Add a subscription + 60-day free trial to Aria so the app can monetize while preserving the local-first trust posture. Phase ships:

1. **Stripe Billing integration** — Customer Portal for self-serve plan/card management, webhook receiver for subscription lifecycle events (created/updated/canceled/payment_failed), single $/month or $/year SKU for v1.
2. **License server** (new external service) — Cloudflare Worker + D1. Receives Stripe webhooks, mints signed entitlement JWTs, validates license keys, issues activation tokens. **NOT in this repo** — separate `aria-license-server` project, deployed independently.
3. **Entitlement client in Aria** — license-key paste UI + `aria://activate?key=...` deep link, periodic online check (24h cadence), Ed25519 JWT verification, 14-day offline grace period, signed-entitlement cache in encrypted SQLite.
4. **60-day trial** — starts on first launch (no card, no signup). Trial state persisted in encrypted SQLite with tamper-resistant signed timestamp from license server captured on first launch (one-shot anonymous registration call).
5. **Paywall UX** — day-61 hard cutoff blocks writes/actions (email send, calendar changes, task push, briefing generation, RAG queries); read-only access to existing data + export remains available. "Subscribe" CTA opens Stripe Checkout in default browser.
6. **Restore-license flow** — email post-checkout delivers key; in-app "Enter license key" field; deep-link handler registers `aria://` URL scheme.

**Out of scope (deferred):**
- Multi-seat / team plans (v1 is single-user)
- Tiered pricing (one paid tier in v1)
- In-app purchase via Mac App Store / MS Store (incompatible with current GitHub Releases updater)
- Promo codes / coupons beyond Stripe defaults
- Refund automation (handled manually via Stripe dashboard)
- Receipt PDFs (Stripe sends these natively)
- Tax compliance beyond Stripe Tax (US/EU sales tax automated via Stripe Tax — confirm enabled at checkout)
- Phase 9 paywall *visual design* — paywall lives in this phase functionally, gets the Anthropic design system applied in Phase 10 (formerly Phase 9 UI)

</domain>

<decisions>
## Implementation Decisions

### Locked (user-confirmed, do not re-litigate)

| Area | Decision |
|---|---|
| **Billing provider** | Stripe Billing — Customer Portal for self-serve, webhooks for lifecycle |
| **Entitlement model** | License key + periodic online check + signed entitlement JWT cached locally |
| **Trial start** | First app launch, no card required, no signup |
| **Trial duration** | 60 days |
| **Phase position** | 08.1 — inserted before Phase 9 (UI) so paywall ships pre-redesign |
| **License server arch** | Cloudflare Worker + D1 (separate `aria-license-server` repo) |
| **Signing scheme** | Ed25519 JWT, 7-day entitlement expiry |
| **Online check cadence** | Every 24h |
| **Offline grace** | 14 days after last successful check before lock |
| **Paywall behavior** | Hard cutoff on day 61; read-only data + export remain available; all writes/actions blocked |
| **Key delivery** | Email key post-Stripe-checkout + in-app paste field + `aria://activate?key=...` deep link |
| **Trial tamper resistance** | Encrypted SQLite + signed `trial_started_at` timestamp from license server on first-launch anonymous registration |

### Derived / Claude's discretion

- **License key format:** ULID-prefixed + checksum, e.g. `ARIA-01HV3...XXXX-K9` (researcher can refine; ULID gives time-ordering for support).
- **Anonymous trial registration:** On first launch, app POSTs to license server with a freshly generated install_id (UUID v4); server returns `{ trial_started_at, signed }`. install_id is NOT a user identifier — never tied to email until purchase.
- **Entitlement JWT claims:** `sub` (license_id), `iat`, `exp` (now + 7d), `tier` ("trial" | "pro"), `install_id`, `features` (string[] for future feature flags). Phase reserves the schema, ships only "trial" and "pro" today.
- **Local entitlement cache:** stored in encrypted SQLite (existing SQLCipher DB from Phase 1), single-row `entitlement` table holding raw JWT + parsed claims + last-verified timestamp. Migration 131 (after Phase 8's 128/129/130).
- **Grace timer behavior:** Each successful online check refreshes `last_verified_at`. Lock triggers when `now - last_verified_at > 14 days`. Lock = same paywall UX as trial expiry.
- **Clock skew defense:** Trust server `iat`/`exp` over local clock when within 24h; if local clock runs >7d behind server-issued JWTs, surface "Check your system clock" notice rather than silent lock.
- **Approval-gate parity:** Paywall lock uses the SAME chokepoint shape as the assertApproved guard from Phases 3/4/6 — a single `assertEntitled()` call wrapped around email_send, calendar_change, task_push, briefing_generate, and rag_query IPC surfaces. Static-grep ratchet enforces.
- **Stripe webhook secret:** stored in Cloudflare Worker env (not in Aria app). App never sees Stripe keys.
- **License server endpoints (v1):**
  - `POST /v1/trial/start` → `{ install_id } → { trial_started_at, jwt }` (anonymous, 60d trial entitlement)
  - `POST /v1/license/activate` → `{ license_key, install_id } → { jwt }` (binds key to install_id, rejects if key bound to ≥3 installs)
  - `POST /v1/entitlement/refresh` → `{ jwt | license_key, install_id } → { jwt }` (24h check)
  - `POST /v1/webhooks/stripe` → handles checkout.session.completed → mints key + emails it
- **Install binding cap:** 3 active installs per license. Self-service revoke in Customer Portal v2 (deferred).
- **Email delivery:** Resend or Postmark from the license server. Recommend Resend (cheap, simple, good deliverability).
- **Restore flow on key loss:** Customer Portal magic link via Stripe → "Resend license key" action calls Worker → re-emails key. No support ticket needed for the happy path.
- **Deep link registration:**
  - macOS: `LSHandlerInfo` in Info.plist via electron-builder `protocols` field
  - Windows: NSIS installer registers `aria://` HKCU URL scheme
  - Linux: `.desktop` MimeType
- **Telemetry stance:** entitlement events (trial start, check success/fail, paywall hit, activation success) logged to local pino only. NO remote telemetry on subscription state — aligns with Phase 8 opt-in-only stance.
- **Privacy disclosure update:** First-launch trial registration sends `install_id` (random UUID) to license server. Must be documented in Trust Center copy added in Phase 8 (TRUST-04).

### Reserved for plan-phase to resolve

- Exact JWT library choice (`jose` vs `@noble/ed25519` + custom JWT) — pick whichever has smallest bundle + fewest native deps.
- Whether Customer Portal session creation goes through license server or directly from app via short-lived Stripe portal session URL.
- Specific paywall UI states (modal vs full-screen takeover) — paywall is functional in 08.1; visual polish lives in Phase 10 UI phase.
- Exact integration point for `assertEntitled()` — researcher should grep existing assertApproved sites and propose same wrapper pattern.

## Specific Ideas

- License server lives in a **separate repo** (`aria-license-server`). Phase 08.1 PLAN.md will scaffold that repo as a sibling project + deploy to Cloudflare in a dedicated plan; Aria itself only ships the client.
- **Three plans projected** (planner will finalize):
  1. **08.1-01 — License server (Cloudflare Worker + D1 + Stripe webhooks + email)** — separate repo scaffold, deployed to staging Cloudflare account. Includes integration test that runs Stripe CLI fixtures end-to-end.
  2. **08.1-02 — Aria entitlement client + trial state + chokepoint** — migration 131, EntitlementService, `assertEntitled()` chokepoint, static-grep ratchet, anonymous trial registration on first launch, 24h check cron via existing node-cron, deep-link handler, license-key paste IPC, encrypted-SQLite cache. Hard-cutoff lock logic. No UI polish.
  3. **08.1-03 — Paywall UX + Customer Portal + restore + activation flow** — functional UI (uses current design system; gets restyled in Phase 10), Stripe Checkout open-in-browser, Customer Portal link, restore-license deep-link e2e, activation success/failure UX, day-50/55/59 in-app trial countdown banner.
- **Test posture:** mock Stripe via Stripe CLI fixtures in CI; mock license server via MSW in Aria's tests. Live OAuth-style smoke test deferred to live release verification (similar to Phase 8 SC pattern).
- **Threat model items for `gsd-secure-phase`:**
  - Local clock manipulation to extend trial → server-signed `trial_started_at` defeats
  - License-key sharing → install_id binding cap of 3
  - Replayed JWTs after revocation → 7d JWT expiry + revocation list on `/v1/entitlement/refresh`
  - SQLCipher DB tampering to flip entitlement → JWT signature verification on every gate check, not just on read
