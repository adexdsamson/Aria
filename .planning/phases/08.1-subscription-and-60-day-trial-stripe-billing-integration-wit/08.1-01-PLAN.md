<action>
Implement the 5 route handlers + a 6th for `/v1/license/resend`. Use Hono. Each route file exports a `Hono` sub-app that gets mounted in `src/index.ts`. Specifications per RESEARCH §4:

- `src/routes/trial.ts` — POST `/v1/trial/start`. Validate body (`install_id: uuidv4`). Call `getOrCreateTrial`. Compute `trial_expires_at = trial_started_at + 60d`. Sign JWT with `{ sub: 'trial:' + install_id, tier: 'trial', install_id, trial_started_at, trial_expires_at, features: [] }`. Return `{ ok: true, trial_started_at, jwt }`.
- `src/routes/license.ts` — POST `/v1/license/activate`. Validate body. Call `verifyChecksum(license_key, env.LICENSE_KEY_SALT)` — return `key-not-found` if invalid format (don't even hit D1). Call `bindInstall`. On success: sign pro JWT, return `{ ok: true, jwt }`. On `install-cap-exceeded` → 409. On other reasons → 400.
- `src/routes/entitlement.ts` — POST `/v1/entitlement/refresh`. Body `{ jwt, install_id }`. Verify JWT. Assert `jwt.install_id === body.install_id` (reject otherwise with `invalid-signature`). Look up license by `sub`. If license.status === 'canceled' → `{ ok: false, error: { code: 'subscription-canceled' } }`. Otherwise issue new JWT with same claims, refreshed iat/exp.
- `src/routes/portal.ts` — POST `/v1/portal/session`. Verify JWT, map to customer, call Stripe billing.portal.sessions.create, return URL.
- `src/routes/webhooks.ts` — POST `/v1/webhooks/stripe`. Verify Stripe signature with `stripe.webhooks.constructEvent(body, sig, env.STRIPE_WEBHOOK_SECRET)`. If event.id already in `webhook_events` → return 200 (idempotent). Otherwise INSERT INTO `webhook_events`. Route on `event.type`:
  - `checkout.session.completed` → newLicenseKey → INSERT licenses → sendLicenseEmail
  - `customer.subscription.updated` → UPDATE licenses SET status, updated_at
  - `customer.subscription.deleted` → UPDATE licenses SET status='canceled'
  - `invoice.payment_failed` → no-op for v1 (rely on dunning); log only
- `src/routes/license.ts` also exposes POST `/v1/license/resend` taking `{ stripe_customer_session_token }` — verify via Stripe.customers.retrieve, look up license, resend email.

`src/index.ts` mounts all six routes, adds CORS (allowed origin `app.aria://` for the few requests originating from Aria — but Aria sends from Node, no Origin header, so CORS is mostly informational), and a `GET /v1/health` returning `{ ok: true, version: 1 }`.
</action>
