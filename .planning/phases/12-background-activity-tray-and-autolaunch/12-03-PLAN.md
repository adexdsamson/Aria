## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Notification click handler -> renderer route push | Main fires webContents.send('aria:navigate', '/briefing') with HARDCODED path; no user-controlled value |
| Preload aria:navigate listener -> renderer router | Path values delivered to renderer; renderer enforces allowlist |
| OS Notification permission state | macOS may deny on first show; fallback is the tray badge |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-10 | Information Disclosure | aria:navigate path used to land on unauthorised renderer route | mitigate | Allowlist ['/briefing', '/approvals'] enforced at App.tsx |
| T-12-11 | Spoofing | Briefing notification fired when briefing is empty/failed | mitigate | showBriefingReadyNotification called ONLY from the run.ts status='done' site |
| T-12-12 | Denial of Service | Notification spam on rapid briefing re-runs | mitigate | _briefingNotificationDedupe Set keyed by briefingDateKey; one notification per day max |
| T-12-13 | Tampering | firstCloseToastShown bypassed by deleting the settings row | accept | Worst case: toast fires twice (cosmetic only); not a security boundary |
